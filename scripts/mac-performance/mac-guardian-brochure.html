AI Engine

Das KI-Gehirn im Detail

Vier KI-Algorithmen arbeiten zusammen fuer autonome Entscheidungen.

📈 1. Adaptive Schwellwerte

Statt fester Grenzwerte berechnet das System dynamische Schwellwerte basierend auf deinem individuellen Nutzungsmuster: Mean + 1.5 × Standardabweichung der letzten Stunde. Was bei dir normal ist, loest keinen Alarm aus.

🔮 2. Vorhersage-Engine

Lineare Regression ueber die letzten 30 Datenpunkte (5 Minuten). Berechnet ob ein kritischer Schwellwert in den naechsten 5–10 Minuten ueberschritten wird — und greift PRAEVENTIV ein.

🚨 3. Anomalie-Erkennung

Z-Score-basiert: Jeder aktuelle Wert wird gegen die statistische Verteilung der letzten Stunde verglichen. Ein Z-Score > 2.5 bedeutet: dieser Wert ist statistisch hochgradig ungewoehnlich.

🔗 4. Korrelations-Analyse

Erkennt zusammenhaengende Probleme und findet die Ursache:

CPU hoch RAM hoch Temp hoch Diagnose Aktion
✅ ✅ — App Overload App identifizieren & beenden
✅ — ✅ Thermal Throttle CPU-Last reduzieren
— ✅ — Memory Pressure Speicher freigeben
✅ ✅ ✅ NOTFALL Emergency-Kill + Kuehlung

🔄 5. Feedback-Loop & Lernen

Nach jeder Aktion wartet das System 30 Sekunden und misst dann, ob die Metriken sich verbessert haben. Erfolgreiche Aktionen werden hoeher gewichtet, wirkungslose werden beim naechsten Mal uebersprungen. Das System wird mit der Zeit besser.

Eskalation

Intelligente Eskalation

Guardian reagiert proportional zum Problem. Nie zu viel. Nie zu wenig.

Memory Guardian — 4 Stufen

Stufe Trigger Aktion Risiko
Soft RAM > 80% System-Cache leeren, Browser-Cache Kein Risiko
Medium RAM > 85% + Erweiterte Caches + Memory Pressure Warning an Apps Kein Risiko
Aggressiv RAM > 90% + Low-Priority Apps beenden (Slack, Spotify...) Apps neu oeffnen
Notfall RAM > 95% + Top 3 RAM-Verbraucher beenden Datenverlust moeglich

Thermal Guardian — 3 Stufen

Stufe Trigger Aktion
Light > 80°C + steigend Spotlight pausieren, Time Machine drosseln
Medium > 90°C + CPU-intensive Hintergrundprozesse drosseln (renice)
Heavy > 95°C + Prozesse temporaer pausieren (SIGSTOP) mit Auto-Resume nach 30s

Process Manager — Schutz-Hierarchie

Prioritaet Apps Behandlung
0 — Gesperrt WindowServer, kernel_task, Finder, Dock Werden NIEMALS angefasst
1 — Kritisch Terminal, Xcode, aktiver Browser Hoechste CPU-Prioritaet
2-3 — Normal Mail, Messages, Dev-Tools Standard-Prioritaet
4-5 — Niedrig Slack, Spotify, Docker, Steam Werden zuerst gedrosselt/beendet

Globale Sicherheitsregeln

  • Max. 5 automatische App-Beendigungen pro Stunde (Schutz vor Endlos-Kill)
  • Geschuetzte Apps werden NIEMALS beendet (konfigurierbar)
  • Vordergrund-App bekommt immer Prioritaet (wird erkannt via AppleScript)
  • Nach jeder Aktion: 30s Evaluation ob sie gewirkt hat
  • Alle Aktionen werden geloggt mit Zeitstempel und Ergebnis
