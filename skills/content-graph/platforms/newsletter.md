# platforms/newsletter.md – Newsletter Content Rules

deep-dive format, value-first, kein Fluff.
expand von [[platforms/linkedin.md]] + [[platforms/youtube.md]].
use [[voice/brand-voice.md]] vollständig.
audience: [[audience/builders.md]] (Kern-Subscriber sind engagierteste Nutzer).

## Format
- Länge: 600–1200 Wörter
- Struktur: Hook → Problem → Lösung → Ressourcen → CTA
- 1 zentrales Thema pro Ausgabe (kein Sammelsurium)

## Sections
```
[SUBJECT LINE] – Problem oder Curiosity Gap
[PREVIEW TEXT] – erster Satz nach Subject Line

[OPENER] – persönliche Geschichte oder aktueller Bezug

[HAUPTTEIL]
- These: 1 Kernidee
- Evidenz: Daten, Beispiel, eigene Erfahrung
- Framework: 3-Schritte oder Checkliste

[RESSOURCEN] – 2–3 weiterführende Links/Tools

[CTA] – eine Handlung (reply, klick, teile)
```

## Posting-Frequenz
1x/Woche (Sonntags 8–10 Uhr) – siehe [[engine/scheduling.md]]
